# AI Spend Audit

AI Spend Audit is a small startup-style MVP built for the Credex internship assignment. It helps a user enter their current AI tool setup, benchmarks the spend against simple plan pricing, highlights obvious overspending, and generates a shareable audit result with a short AI-written summary.

The project is intentionally simple. The goal was not to build an enterprise finance platform, but to ship something realistic, understandable, and interview-defensible within a 7-day timeline.

## What it does

- Collects AI tool, plan, monthly spend, seats, team size, and use case
- Persists in-progress form state in `localStorage`
- Runs deterministic audit logic in `lib/audit/`
- Shows current spend, benchmark spend, recommendation, monthly savings, and annual savings
- Generates a short personalized summary using the Anthropic API with a fallback summary
- Saves audits and leads to Supabase when environment variables are available
- Exposes shareable result routes at `/result/[id]`

## Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase
- Anthropic API
- Vitest

## Local setup

1. Install dependencies:

```bash
npm install
```

2. Create `.env.local`:

```bash
NEXT_PUBLIC_SUPABASE_URL=your-supabase-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key
ANTHROPIC_API_KEY=your-anthropic-api-key
```

3. Run the app:

```bash
npm run dev
```

4. Run tests:

```bash
npm run test -- --run
```

5. Run lint:

```bash
npm run lint
```

## Supabase tables

`audits`

- `id` text primary key
- `tool_name` text
- `current_plan` text
- `monthly_spend` numeric
- `seats` numeric
- `team_size` numeric
- `use_case` text
- `audit_payload` jsonb

`leads`

- `id` uuid default generated by Supabase
- `audit_id` text
- `name` text
- `email` text
- `company` text

## Product decisions

- I kept the audit engine deterministic so every recommendation can be explained line by line.
- I used AI only for the final summary paragraph, not for calculations or pricing logic.
- I saved audit results in Supabase when possible, but also cache the latest results in `localStorage` so the flow still works during local development.
- I chose a clean, minimal UI because the assignment benefits more from clarity than from visual complexity.
