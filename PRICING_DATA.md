## Why the pricing data is simple

I kept pricing data intentionally small because:

- the assignment is about believable product thinking, not full procurement coverage
- a beginner-to-intermediate developer should be able to explain every price row
- deterministic recommendations are easier to defend when the data model is small

## Limitations

This pricing model does not include:

- annual discounts
- usage-based API billing
- credits
- enterprise custom quotes
- region-specific pricing
- feature-level add-ons

A real product would need those, but they would add complexity without helping this assignment much.

## How the app uses pricing

Pricing is used for:

- computing benchmark spend
- comparing current plan against simpler alternatives
- estimating monthly and annual savings

Pricing is not generated by AI. It is stored directly in code and used by deterministic TypeScript functions.
