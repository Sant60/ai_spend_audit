# Metrics

## Core product metrics

For this MVP, I would track a small set of metrics that match the actual flow.

### Funnel metrics

- landing page visitors
- audit starts
- audit completions
- result page views
- lead form submissions

### Quality metrics

- average estimated monthly savings
- average estimated annual savings
- percentage of audits with at least one overspend finding
- percentage of audits that recommend downgrading the same tool
- percentage of audits that recommend switching tools

### Conversion metrics

- visitor to audit-start conversion
- audit-start to report-complete conversion
- report-complete to lead-submit conversion

## Success signals

Good early signs would be:

- users complete the audit without confusion
- savings numbers are large enough to feel worth discussing
- lead submissions come from reports with clear overspend findings

## Metrics I would avoid at this stage

- vanity social reach metrics
- broad engagement dashboards
- deep cohort slicing before traffic exists

This product is still too early for that.

## Why this metric set is enough

The app only has one real job:

1. get someone to run an audit
2. show a believable result
3. convert a percentage of those users into leads

These metrics line up directly with that job.
